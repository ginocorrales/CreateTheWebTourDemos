About

CSS Regions and CSS Exclusions bring new properties to CSS (Cascading Style Sheets) that provide:

  • text containers with custom shapes.
  • exclusion shapes which text will wrap around.
  • text that flows from one area into another.

We have put together a set of demos for you. They showcase some of the concepts we proposed to the W3C with CSS Regions and CSS Exclusions: content threads, content shapes and text exclusions.

The samples presented here require a mini-browser using a specially modified version of WebKit. This experimental version will allow you to experiment with the samples, and are a way for us to explore the implementation options for these features. Please see the readme.txt file for instructions on how to run this browser.

You'll find basic samples that demonstrate individual CSS properties, as well as more complex ideas that highlight how they work together.

Why use CSS Regions

Web Designers and Developers

  • Take advantage of new CSS properties that let you build rich interfaces with custom-shaped content boxes, text exclusions and multiple text flows.
  • CSS Regions are easy to pick up. They build upon familiar CSS properties and structure.

Basic samples

  • Simple content thread

     

    Demonstrates: text flowing from one region into another.

    CSS properties used: -webkit-flow, content: from()

  • Content order

     

    Demonstrates: text flowing through regions in a custom-defined order.

    CSS properties used: -webkit-flow, -webkit-content-order, content: from()

  • Multiple content threads

    Demonstrates: multiple content threads with text flowing through regions of varying shapes.

    CSS properties used: -webkit-flow, content: from()

  • Content shape

     

    Demonstrates: custom shape for text container.

    CSS properties used: -webkit-wrap-shape, -webkit-wrap-shape-mode

  • Text exclusion

     

    Demonstrates: exclusion shape affecting text layout.

    CSS properties used: -webkit-wrap-shape, -webkit-wrap-shape-mode

  • Dynamic exclusion

     

    Demonstrates: text flowing around an exclusion.

    CSS properties used: -webkit-wrap-shape, -webkit-wrap-shape-mode

  • Shape order

     

    Demonstrates: shapes affecting each other

    CSS properties used: -webkit-wrap-shape, -webkit-wrap-shape-mode, -webkit-wrap-shape-order

  • Dropcap exclusion

     

    Demonstrates: exclusions and regions working together to create a selectable multicharacter dropcap.

    Tip: resize the browser window to see how the text flows next to the dropcap.

    CSS properties used: -webkit-wrap-shape, -webkit-wrap-shape-mode, -webkit-flow, content: from()

    Tip: Switch between device screens and orientations to see how the layout can adapt to changes in size.

  • Into The Great Wide Open

    Tip: Notice how the text avoids the top and bottom illustrations. Switch between device screens and orientations to see how the layout can adapt to changes in size.

Feedback

Please keep in mind that the CSS Regions prototype is a work in progress.

The development so far has been done following the the CSS Regions and the CSS Exclusions editor's drafts. Comments and suggestions for these documents are very welcome!

  • For comments on the CSS Regions editor's draft proposal please use the www-style@w3.org mailing list on the W3C website. Please use an identifier such as "[CSS Regions]" in the subject of your email.
  • For comments on the CSS Exclusions editor's draft proposal please use the www-style@w3.org mailing list on the W3C website. Please use an identifier such as "[CSS Exclusions]" in the subject of your email.
  • If you have any questions or comments about the current prototype or the samples please drop us a message on the CSS Regions forum.

Any feedback on the proposals is very welcome. It helps us provide a feature that's both fun and useful in the modern web landscape.
