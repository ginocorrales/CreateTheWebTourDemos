Threading multiple content flows with CSS Regions

The content of an element can be broken into multiple named flows. These flows can be associated to different region chains. This way the layout of the content is no longer bound to its structure.

Consuming from named flows

Bits of content from named flow can pulled into another named flow.

Child nodes beloging to an element associated to a named flow can be pulled into a separate named flow using a more specific selector and the flow-into property.

Code Sample

article{
    flow-into: article;
}

article aside{
    flow-into: secondary;
}

The content of the aside is pulled from the 'article' flow into the 'secondary' flow

Content is never duplicated. The content targeted with the more specific selector is taken out of the first flow and moved to the other flow.
