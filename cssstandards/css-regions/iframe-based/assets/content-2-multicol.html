The multi-column model

Multi-column elements flow their contents into column boxes. This contrasts with the traditional

Usually, columns are ordered in a single row, according to the inline direction of the element. In the case of paginated media, a new row is created for the new page – since column boxes should not break.

CSS model where each element has a box in which the contents are laid out.

While traditionally the children of a multi-column elements where flowed into columns, new CSS specs (e.g. CSS regions) enhance the type of content that can be flown into multiple columns.

Currently, individual columns cannot be styled; for instance, the background-color cannot be changed for a single column inside a multi-column element.

How to achieve multi-column?

An element becomes multi-column either of the colum-count or column-width CSS properties are not auto. Specifying a value for the any of these properties turns the element into a multi-column element. Further use of CSS properties can specify how column should be spaced, whether or not they should be visually separated, whether the content should be balanced between columns and whether or not columns should be automatically generated.
